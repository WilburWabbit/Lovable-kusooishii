## Msgs 448–450 — FIFO duplicate stock unit fix
Diagnosed: 9 receipt lines with excess stock units. Two bugs:
1. Race condition in qbo-webhook handlePurchase — duplicate QBO notifications both see status=pending, both delete+recreate lines and insert units.
2. No shortfall guard in either `handlePurchase` or `process-receipt`.
Implemented shortfall check in both functions:
```typescript
const { count: existingCount } = await admin
  .from("stock_unit")
  .select("id", { count: "exact", head: true })
  .eq("inbound_receipt_line_id", line.id);
const shortfall = line.quantity - (existingCount ?? 0);
if (shortfall > 0) { /* insert only shortfall units */ }
```
Cleanup migration: deleted excess units per line, keeping oldest N where N = expected quantity.
