# Unified Product Specifications + Channel Mapping Registry\n\n## Goals (re-stated to confirm understanding)\n\n1. **One Specifications tab** per product. No duplicate manual entry of the same fact in multiple places. Each fact is captured **once** — either:\n   - directly on the product (e.g. `age_mark`, `dimensions_cm`, `weight_g`, `ean`), or\n   - sourced from a **canonical provider** the app already has (BrickEconomy, Rebrickable, `lego_catalog`, BrickLink, BrickOwl).\n2. **eBay category lives on the Specifications tab.** Searching/selecting an eBay category drives **which fields the tab displays for editing**, because the displayed list is exactly the union of:\n   - fields the app always tracks (identity / lifecycle / physical), plus\n   - any extra aspects required by the chosen eBay category that are **not already mapped** to a canonical source.\n3. **Mapping is backend-owned and database-driven.** A new Settings page lets you maintain, per eBay category aspect:\n   - which canonical attribute (or constant) it maps to, and\n   - the priority order of providers (e.g. manual override → product column → BrickEconomy → catalog).\n4.
